Numerical Optimization for Symmetric Tensor Decomposition

Posted on 2022-12-01 In paper

Basic Information

  • Title: Numerical Optimization for Symmetric Tensor Decomposition
  • Authors: Tamara G. Kolda et al.
  • Pub Date: 2015-06-01
  • Journal/Conference: Mathematical Programming

Introduction

  • For symmetric real-valued, \(m\)-way and \(n\)-dimension tensor \(\boldsymbol{\mathcal{A}}\), it can be decomposed as \[ \boldsymbol{\mathcal{A}} = \sum\limits_{k=1}^{p} \lambda_{k} \boldsymbol{x}_{k}^{m}, \tag{\S} \]

    with \(\lambda_{k}\in \mathbb{R}\) and \(\boldsymbol{x}_{k}\in\mathbb{R}^n\) for \(k = 1, \dots, p\).

  • Let \(\boldsymbol{\mathcal{A}} \ge 0\) represent an \(m\)-way, \(n\)-dimension nonnegative symmetric tensor. In this case, the goal is the factorization of a the form \[ \boldsymbol{\mathcal{A}} = \sum\limits_{k=1}^{p} \boldsymbol{x}^{m}_{k} \quad \text{with}~\boldsymbol{x}_{k}\ge 0 \] If such a factorization exists, we say that \(\boldsymbol{\mathcal{A}}\) is completely positive.

CP Decomposition

Details: - Refer to Tensor Decomposition and Applications

CP Decomposition with Partial Symmetry

For simplicity of discussion, we assume a cubic tensor \(\boldsymbol{\mathcal{A}} \in \mathbb{R}^{[m, n]}\). For \(m = 3\) and the last two dimensions being symmetric, this means \[ a_{ijk} = a_{ikj} \quad \forall i, j, k \in \{ 1, \dots, n \}, \] and the factorization should be of the form \[ \boldsymbol{\mathcal{A}} = \sum\limits_{k=1}^{p} \boldsymbol{u}_{k} \circ \boldsymbol{v}_{k} \circ \boldsymbol{v}_{k} \] In other words, the last two vectors in each component are equal.

  • Carroll and Chang proposed to use an alternating method that ignores symmetry, with the idea that it will often converge to a symmetric solution (up to diagonal scaling).
  • Later work showed that not all KKT points satisfy this condition

Best Symmetric Rank-1 Approximation

\[ \min \| \boldsymbol{\mathcal{A}} - \lambda \boldsymbol{x}^{m} \| \quad \text{subject to}~ \lambda \in \mathbb{R},~\boldsymbol{x} \in \mathbb{R}^{n}. \]

This problem is directly related to the problem of computing tensor Z-eigen pair. A pair \((\lambda, \boldsymbol{x})\) is a Z-eigen pair of a tensor \(\boldsymbol{\mathcal{A}} \in \mathbb{S}^{[m, n]}\) if

\[ \boldsymbol{\mathcal{A}} \boldsymbol{x}^{m-1} = \lambda \boldsymbol{x} ~\text{and}~ \|\boldsymbol{x}\| = 1. \] where \(\boldsymbol{\mathcal{A}}\boldsymbol{x}^{m-1}\) denotes a vector in \(\mathbb{R}^n\) such that \[ (\boldsymbol{\mathcal{A}} \boldsymbol{x}^{m-1})_{i_{1}}= a_{\boldsymbol{i}} x_{i_{2}} \cdots x_{i_{m}} ~\text{for}~ i_{1} \in \{1, \dots, n\}. \]

The problems are related because any KKT point is a Z-eigen pair of \(\boldsymbol{\mathcal{A}}\).

Symmetric Tucker Decomposition

Here the goal is to find an orthogonal matrix \(\boldsymbol{U} \in \mathbb{R}^{n \times p}\) and a symmetric tensor \(\boldsymbol{\mathcal{B}} \in \mathbb{S}^{[m, p]}\) that solves \[ \begin{aligned} &\min \left\| \boldsymbol{\mathcal{A}} - \hat{\boldsymbol{\mathcal{A}}} \right\|\\ &\text{s.t.}~\hat{a}_\boldsymbol{i} = b_{j_{1}j_{2}\dots j_{m}} u_{i_{1}j_{1}} u_{i_{2}j_{2}} \cdots u_{i_{m}j_{m}} \end{aligned} \]

Complex-valued symmetric tensor decomposition

\[ \boldsymbol{\mathcal{A}} = \sum\limits_{k=1}^{p} \boldsymbol{x}_{k}^{m} \quad \text{with}~ \boldsymbol{x}_{k} \in \mathbb{C}^{n}~(k=1, \dots, p) \]

Optimization Formulations for Symmetric Tensor Decomposition

Index Multiplicities

For \(\boldsymbol{\mathcal{A}} \in \mathbb{S}^{[m, n]}\) has \(n^{m}\) entries.

  1. Let the set of all possible indices be denoted by \[ \mathcal{R} = \{(i_{1}, \dots, i_{m}) \mid i_{1}, \dots, i_{m} \in \{ 1, \dots, n \}\} \]

    Obviously, \(|\mathcal{R}| = n^m\).

  2. Following, we define an index class as a set of tensor indices such that the corresponding tensor entries all share a value due to symmetry. \[ \mathcal{I} = \{(i_{1}, \dots, i_{m}) \mid i_{1}, \dots, i_{m} \in \{ 1, \dots, n \} ~\text{and}~i_{1}\le i_{2}\le \cdots \le i_{m}\} \subset \mathcal{R} \]

  3. Each index class also has a monomial representation. For each \(\boldsymbol{i} \in \mathcal{I}\) there exists a corresponding monomial representation \(\boldsymbol{c}\) such that \[ x_{i_{1}} x_{i_{2}} \cdots x_{i_{m}} = x_{1}^{c_{1}}x_{1}^{c_{2}}\cdots x_{n}^{c_{n }} \] Specially, \(c_{j}\) represents that number of occurrences of index \(j\) in \(\boldsymbol{i}\) of \(j = 1, \dots, n\). Clearly, \(\sum\limits_{j} c_{j} = m\). Conversely, for a given, \(\boldsymbol{c}\), we build an index \(\boldsymbol{i}\) with \(c_1\) copies of \(1\), \(c_{2}\) copies of \(2\), etc. The set of monomial representations is denoted by \[ \mathcal{C} = \{ (c_{1}, \dots, c_{n}) \mid c_{1}, \dots, c_{n} \in \{ 0, \dots, m \} ~\text{and}~ c_{1} + \dots + c_{n} = m \}. \] And \[ |\mathcal{I}| = |\mathcal{C}| = C_{m}^{m+n-1} = \frac{n^{m}}{m!} + O(n^{m-1}) \]

  4. The multiplicity of the entry corresponding to a monomial representation \(\boldsymbol{c}\) is \[ \sigma_\boldsymbol{c} = \begin{pmatrix} m \\ c_{1}, c_{2}, \dots, c_{n}\end{pmatrix} = \frac{m!}{c_{1}! c_{2}!\cdots c_{n}!} \]

For \(\mathcal{S}^{[3, 2]}\):

In this case \[ \|\boldsymbol{\mathcal{A}}\|^{2}= \sum\limits_{\boldsymbol i \in \mathcal R} a_{\boldsymbol{i}}^{2}= \sum\limits_{\boldsymbol{i} \in \mathcal{I}} \sigma_\boldsymbol{i} a_\boldsymbol{i}^{2} = \sum\limits_{\boldsymbol{c} \in \mathcal{C}}\sigma_\boldsymbol{c} a_\boldsymbol{c}^{2}, \] and \[ (\boldsymbol{x}_{k}^{m})_{\boldsymbol{i}} = x_{i_{1}k} x_{i_{2}k} \cdots x_{i_{m}k} = (\boldsymbol{x}_{k}^{m})_\boldsymbol{c} = x_{1k}^{c_{1}} x_{2k}^{c_{2}} \cdots x_{nk}^{c_n} \]

Two Formulations

For given \(\boldsymbol{\mathcal{A}} \in \mathbb{R}^{[m,n]}\) and \(p\), our goal is to find \(\boldsymbol{\lambda}\) and \(\boldsymbol{X}\) such that \((\S)\) is satisfied in a minimization sense.

  1. Standard Least Squares \[ f_{1}(\boldsymbol{\lambda}, \boldsymbol{X}) = \sum\limits_{\boldsymbol{i} \in \mathcal{R}}\left( a_\boldsymbol{i} - \sum\limits_{k=1}^{p} \lambda_{k} (\boldsymbol{x}_{k}^{m})_\boldsymbol{i} \right)^{2} = \sum\limits_{\boldsymbol{i} \in \mathcal{I}} \sigma_\boldsymbol{i} \left( a_\boldsymbol{i} - \sum\limits_{k=1}^{p} \lambda_{k} (\boldsymbol{x}_{k}^{m})_\boldsymbol{i} \right), \]

    This counts each unique entry multiple times, according to its multiplicity.

  2. Counts each unique entry only once \[ f_{2}(\boldsymbol{\lambda}, \boldsymbol{X}) = \sum\limits_{\boldsymbol{i} \in \mathcal{I}} \left( a_\boldsymbol{i} - \sum\limits_{k=1}^{p} \lambda_{k} (\boldsymbol{x}^{m}_{k})_\boldsymbol{i} \right)^2 \]

Both of them can be expressed as \[ f_{\boldsymbol{w}}(\boldsymbol{\lambda}, \boldsymbol{X}) = \sum\limits_{\boldsymbol{i} \in \mathcal{I}} {w}_\boldsymbol{i} \left( a_\boldsymbol{i} - \sum\limits_{k=1}^{p} \lambda_{k} (\boldsymbol{x}^{m}_{k})_\boldsymbol{i} \right)^{2} = \sum\limits_{\boldsymbol{i} \in \mathcal{I}} w_\boldsymbol{i} \delta_\boldsymbol{i}^{2}. \] - \({w}_\boldsymbol{i} = \sigma_\boldsymbol{i}\) yields \(f_1\) whereas \(w_\boldsymbol{i} =1\) yields \(f_{2}\) - \(\delta_\boldsymbol{i}\) denotes the difference between the model and the tensor at entry \(\boldsymbol{i}\).

Penalty

  • Scaling Ambiguity: require \(\|\boldsymbol{x}_{k}\| = 1\) \[\begin{aligned} p_{\gamma}(\boldsymbol{X}) &= \gamma \sum\limits_{k=1}^{p}\left( \boldsymbol{x}_{k}^{\top}\boldsymbol{x}_{k} -1 \right)^{2}\\ \Rightarrow \frac{\partial p_{\gamma}}{\partial \boldsymbol{x}_{k}}&= 4\gamma \left( \boldsymbol{x}_{k}^{\top} \boldsymbol{x}_{k} -1 \right) \boldsymbol{x}_{k} \end{aligned} \]

  • Sparse Ambiguity: when \(p\) (rank) is large. use an approximate \(l_{1}\) penalty \[ p_{\alpha, \beta}(\boldsymbol{\lambda}) = \frac{\beta}{\alpha} \sum\limits_{k=1}^{p} \left\{\log [1 + \exp(-\alpha \lambda_{k})] + \log [1 + \exp(\alpha \lambda_{k})]\right\} \approx \beta \|\boldsymbol{\lambda}\|_{1} \] Note that the \(\beta\) term is not part of the approximation but rather the weight of the penalization.

Above all, the final function to be optimized is \[ \hat{f}(\boldsymbol{\lambda}, \boldsymbol{X}) = f_\boldsymbol{w}(\boldsymbol{\lambda}, \boldsymbol{X}) + p_{\gamma} (\boldsymbol{X}) + p_{\alpha, \beta} (\boldsymbol{\boldsymbol{\lambda}}) \] - \(\boldsymbol{w}\) determines the choice of objective function - \(\gamma\) determines the weight of the penalty on the norm of the columns of \(\boldsymbol{X}\) - Since this constraint is easy to satisfy and mostly convenience, the exact choice of \(\gamma\) is not critical. - \(\alpha\) determines the steepness of the \(l_{1}\) penalty function. - \(\beta\) determines the weight of the sparsity-encouraging penalty.

Ignoring Symmetry

Basic Idea: - Under mild conditions, the CP decomposition is unique up to permutation and scaling of the components, i.e., essentially unique. - The sufficient condition for the uniqueness of CP decomposition: \[ 2p + (m-1) \le m~\text{k-rank}(\boldsymbol{X}) \tag{\dagger} \] - The \(\text{k-rank}(\boldsymbol{X})\) is the largest number \(k\) such that every subset of \(k\) columns of \(\boldsymbol{X}\) is linearly independent. - \(\text{k-rank}(\boldsymbol{X}) \le \min\{n, p\}\) - The importance of essential uniqueness is that the global solution of the unconstrained problem is the same as for the symmetric problem so long as X satisfies \((\dagger)\).

Advantage: We can use existing software for the CP decomposition.

Disadvantage: It requires \(m\) times as much storage.
